---------------------------------------------------------
SECTION 1: WHAT IS THIS PROJECT?
---------------------------------------------------------

This is a BACKEND SERVER for a Smart Agriculture Assistant App.

- It helps FARMERS with agriculture-related questions.
- It is built using NODE.JS + EXPRESS + MONGODB.
- It supports MARATHI LANGUAGE for Indian farmers.
- It runs on PORT 5001 on the computer.
- The frontend (mobile/web app) talks to this backend using API calls.

Think of it like this:
   Farmer App (Frontend) --> sends request --> Backend Server --> gives answer back


---------------------------------------------------------
SECTION 2: TECHNOLOGIES USED
---------------------------------------------------------

1. Node.js       = The runtime environment (like an engine to run JavaScript on server)
2. Express.js    = Web framework (handles HTTP requests and routes)
3. MongoDB       = Database (stores all data like experts, stores, soil data)
4. Mongoose      = Tool to connect Node.js to MongoDB easily
5. Dotenv        = Reads secret settings from .env file
6. CORS          = Allows frontend app to connect to this backend
7. Multer        = Handles image file uploads (for plant disease photos)
8. Axios         = Makes HTTP requests to external APIs (like weather)
9. Nodemon       = Auto-restarts server when code changes (for development)


---------------------------------------------------------
SECTION 3: HOW TO START THE SERVER
---------------------------------------------------------

Step 1: Open terminal in the project folder
Step 2: Run this command:

        npm run dev       (for development - auto-restarts on changes)
        OR
        npm start         (for production - normal start)

Step 3: Server starts on:  http://localhost:5001
Step 4: Test it by opening: http://localhost:5001/health


---------------------------------------------------------
SECTION 4: MAIN FILE - server.js
---------------------------------------------------------

This is the MOST IMPORTANT FILE. It does:

1. Starts the Express web server on Port 5001
2. Connects to MongoDB database (mongodb://localhost:27017/smartAgri)
3. Loads all route files (crop, weather, pest, etc.)
4. Sets up CORS (allows any website to connect)
5. Has a health check endpoint: GET /health
6. Has error handling (handles crashes, port conflicts)
7. Gracefully shuts down when stopped (Ctrl+C)

Important Settings in server.js:
- PORT = 5001 (can change in .env file)
- MongoDB = connects to local database
- If MongoDB fails = server still runs with mock data (no crash)

SECTION 6: MODELS FOLDER (7 Files)
---------------------------------------------------------

Models = Database Table Definitions (what data looks like)
Location: /models/

1. Expert.js
   - Stores agricultural expert information
   - Fields: name, specialization, phone, email, location, availability, rating
   - Example: Dr. Ramesh - Soil Expert - Available Mon-Fri

2. ExpertRequest.js
   - Stores when a farmer requests to talk to an expert
   - Fields: farmerName, phone, expertId, problem description, status, date
   - Status can be: pending, approved, completed, cancelled

3. PestDetection.js
   - Stores pest detection results
   - Fields: image path, pest name, severity, treatment suggestions, date

4. Query.js
   - Stores farmer questions/queries
   - Fields: question text, answer, language, date

5. SoilAnalysis.js
   - Stores soil test results
   - Fields: pH level, nitrogen, phosphorus, potassium, moisture, recommendations

6. Store.js
   - Stores agriculture shop information
   - Fields: store name, address, phone, location coordinates, products, category

7. SystemConfig.js
   - Stores system-level settings
   - Fields: config key, config value, description

SECTION 7: ROUTES FOLDER (13 Files)
---------------------------------------------------------

Routes = API Endpoints (URLs that the frontend calls)
Location: /routes/

1. cropRoutes.js --> /api/crop
   - POST /api/crop/recommend = recommends crops based on soil and weather
   - Input: soil type, location, season
   - Output: list of recommended crops

2. fertilizerRoutes.js --> /api/fertilizer
   - POST /api/fertilizer/recommend = recommends fertilizers
   - Input: crop type, soil analysis results
   - Output: fertilizer name, quantity, schedule

3. soilRoutes.js --> /api/soil
   - POST /api/soil/analyze = analyzes soil data
   - Input: soil test values (pH, NPK)
   - Output: soil health status and recommendations

4. pestRoutes.js --> /api/pest
   - POST /api/pest/detect = detects pests from description
   - Input: symptoms, crop type
   - Output: pest name, treatment, prevention tips

5. diseaseRoutes.js --> /detect
   - POST /detect = detects plant disease from IMAGE
   - Input: plant photo (uploaded image)
   - Output: disease name, severity, treatment
   - GET /detect/list = list of all known diseases

6. weatherRoutes.js --> /api/weather
   - GET /api/weather/:location = gets weather data
   - Input: city/village name
   - Output: temperature, rainfall, humidity, farming advice

7. marketRoutes.js --> /api/market
   - GET /api/market/prices = current crop market prices
   - Output: crop name, price per quintal, market location

8. aiRoutes.js --> /api/ai
   - POST /api/ai/chat = AI-powered chat response
   - Input: user message/question
   - Output: AI-generated farming advice

9. chatbotRoutes.js --> /api/chatbot
   - POST /api/chatbot/chat = Chatbot conversation
   - POST /api/chatbot/advice = Get specific advice
   - GET /api/chatbot/tips = Get farming tips of the day

10. expertRoutes.js --> /api/expert
    - GET /api/expert/list = list all available experts
    - GET /api/expert/:id = get one expert's details
    - POST /api/expert/contact = send message to expert
    - POST /api/expert/book-consultation = book a meeting
    - GET /api/expert/:id/availability = check expert's free slots

11. expertRequestRoutes.js --> /api/expert-request
    - POST /api/expert-request/request = submit a new expert request
    - GET /api/expert-request/request/:requestId = check request status
    - POST /api/expert-request/request/:requestId/cancel = cancel a request
    - GET /api/expert-request/requests/all = view all requests (admin)

12. storeRoutes.js --> /api/stores
    - POST /api/stores/nearby = find stores near a location
    - GET /api/stores/:id = get one store's details
    - GET /api/stores/categories/list = list store categories

13. systemConfigRoutes.js --> /api/system
    - POST /api/system/init-config = initialize system settings
    - GET /api/system/config = get current settings
    - PUT /api/system/update-config = update settings

SECTION 16: COMMON PROBLEMS & SOLUTIONS
---------------------------------------------------------

PROBLEM 1: MongoDB not connecting
   SOLUTION: Make sure MongoDB is running
   - Open Services (Windows) and start MongoDB
   - Or run: mongod in a new terminal

PROBLEM 2: Port 5001 already in use
   SOLUTION: Run kill-port.bat
   - Or run: netstat -ano | findstr :5001
   - Then: taskkill /F /PID <number>

PROBLEM 3: Server crashes immediately
   SOLUTION: Check .env file has correct settings
   - Make sure MONGODB_URI is correct

PROBLEM 4: No data in database
   SOLUTION: Run the init file
   - node init-db-native.js

PROBLEM 5: CORS error in browser
   SOLUTION: Already handled! CORS is enabled for all origins in server.js
